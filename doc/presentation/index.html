Платформа «ЕГЭ Прорыв»

Проблема

  • Платформы ЕГЭ подбирают задания, но не адаптируют теорию.
  • Ученик с устойчивыми ошибками собирает объяснение сам — из видео, чатов, заметок.
  • LLM генерируют связный текст, но без ограничений подвержены галлюцинациям.

Цель и задачи

Цель: разработать сервис, автоматически формирующий персонализированный конспект по математике ЕГЭ на основе цифрового следа ученика.
  1. Определить состав аналитики для персонализации: ошибки, прогресс, mastery.
  2. Спроектировать RAG-пайплайн с предметной базой знаний.
  3. Реализовать многошаговую генерацию с верификацией.
  4. Интегрировать сервис в действующую платформу «ЕГЭ Прорыв».
  5. Оценить качество конспектов: персонализация, корректность, читаемость.

Архитектура сервиса

Предметная база знаний

Вместо автоматического chunking — экспертно размеченные атомы:

  • id — идентификатор (t06_mm01, t12_mm03, …)
  • text — учебный текст с формулами и примерами
  • error_tags — теги типичных ошибок
  • prerequisites — DAG пререквизитов
  • shared_skills — межзадачные навыки
+ файл misconceptions.yaml — экспертные описания типичных заблуждений

DAG + shared skills

RAG-пайплайн

Гибридный RRF: RRF(d) = wdense / (k + rdense) + wsparse / (k + rsparse)
Профильный буст: S(d) = Sfused(d) + γ · (err(d) + β · skill(d)) − 𝟙[m>θ] · δ

Векторное представление и гибридный поиск

Dense — mxbai-embed-large-v1 (1024d)
Семантическая близость тем и понятий
Sparse — BM25 (Qdrant/bm25)
Точное совпадение ключевых терминов
Qdrant — два именованных пространства
payload-индексы по task_number, subtypes → серверная фильтрация
Инкрементальное обновление: SHA-256 хеш контента → пропуск повторного эмбеддинга

Генерация и верификация

Промпт включает:

  • Задание + подтип
  • Уровень детализации
    (по точности: <30% → подробный)
  • Топ-5 тегов ошибок
  • Извлечённый контекст
  • Темы-фронтиры (зона ближайшего развития)
Постобработка: удаление ID атомов, блоков мышления, принудительная конвертация JSON → Markdown

Научно-прикладная новизна

  1. Интеграция learner modeling + RAG + LLM в едином прикладном контуре.
  2. Генерация полноценного учебного артефакта, а не отдельного ответа в чате.
  3. Воспроизводимый пайплайн с явными этапами и настраиваемыми параметрами.
  4. Подход расширяем на другие предметы и форматы экзаменов.
